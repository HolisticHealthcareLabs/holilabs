The Manifesto

Why we're building this

The System is Broken

Modern healthcare infrastructure is collapsing. $4.1 trillion spent annually in the US alone. 90% of that goes to managing chronic, preventable diseases. Clinicians spend more time on paperwork than patients. Data is siloed across incompatible systems. Patients have no control.

Reactive, Not Proactive

The current system is architected for late-stage interventions. Wait until someone is sick, then intervene with expensive treatments. This is economically perverse and medically bankrupt. Prevention is 10x cheaper and infinitely more humane.

Technology Enables the Shift

AI can eliminate administrative burden. Interoperable protocols (FHIR) can unify fragmented data. Blockchain can provide verifiable, portable health records. Web3 primitives can return data ownership to patients. The tools exist. The infrastructure doesn't.

Holi Protocol is the Infrastructure

We're building the missing layer: a proactive, prevention-first, standards-based protocol. It restores clinician time through AI automation. It unifies patient data through FHIR compliance. It empowers patients with verifiable, portable records. It shifts incentives from managing sickness to cultivating health.

This is not a product. This is a protocol — built on open standards and designed for interoperability.

Protocol Architecture

Open-source stack designed for interoperability and sovereignty

Application Layer

  • → Next.js 14 (React)
  • → Tailwind CSS
  • → Progressive Web App
  • → Offline-first architecture

AI/Automation Layer

  • → Whisper (speech-to-text)
  • → Llama 3.1 (clinical reasoning)
  • → FastAPI backend
  • → Self-hosted inference

Data Layer

  • → Medplum (FHIR R4 server)
  • → PostgreSQL + Prisma
  • → Row-level security (RLS)
  • → End-to-end encryption

Integrity Layer

  • → ethers.js (blockchain)
  • → Polygon/Arbitrum (L2)
  • → Veramo (DIDs + VCs)
  • → IPFS (document storage)

Compliance Layer

  • → HIPAA Safe Harbor de-ID
  • → Differential privacy (ε/δ)
  • → Open Policy Agent (OPA)
  • → Audit logging

Infrastructure

  • → Docker + Kubernetes
  • → Redis (caching/queues)
  • → MinIO (S3-compatible)
  • → Self-hostable
