RESTful API with RBAC enforcement and comprehensive health checks.

## Authentication

All routes require `Authorization: Bearer <token>`. Tokens issued by NextAuth.js carry tenant ID, user role, and session metadata.

## Route Protection

Every route uses `createProtectedRoute`:

```typescript
import { createProtectedRoute } from '@/lib/auth';

export const GET = createProtectedRoute({
  roles: ['ADMIN', 'CLINICIAN'],
  handler: async (req, ctx) => {
    // Route logic
  },
});
```

## Core Endpoints

### Safety
- `POST /api/prescriptions/safety-check` ‚Äî unified clinical + financial pre-check
- `GET /api/encounters/[id]/summary` ‚Äî encounter + prescriptions + counts

### Encounters
- `PATCH /api/encounters/[id]/status` ‚Äî state machine: SCHEDULED‚ÜíCHECKED_IN‚ÜíIN_PROGRESS‚ÜíCOMPLETED
- `POST /api/encounters/[id]/prescriptions` ‚Äî inline safety checks; RED‚Üí422; AMBER‚Üí201+warning

### Patient Data Rights (LGPD)
- `GET /api/patients/[id]/export` ‚Äî full Habeas Data export
- `POST /api/patients/[id]/deletion-request` ‚Äî initiate erasure request
- `POST /api/patients/[id]/erasure` ‚Äî cryptographic key destruction

### WebAuthn
- `POST /api/auth/webauthn/register-options`
- `POST /api/auth/webauthn/verify-registration`
- `POST /api/auth/webauthn/sign-options`
- `POST /api/auth/webauthn/verify-signature`

### Health
- `GET /api/health` ‚Äî general health
- `GET /api/health/live` ‚Äî liveness probe
- `GET /api/health/ready` ‚Äî readiness probe

## Audit Middleware

All mutative routes fire an audit event via fire-and-forget buffer (`lib/api/audit-buffer.ts`). The buffer prevents audit overhead from blocking response latency.
